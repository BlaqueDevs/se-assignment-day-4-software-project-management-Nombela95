# SE_DAY4_Software-Project-Management
## 1. Why is timely delivery crucial in software project management, and how can project managers ensure that deadlines are met?
Timely delivery ensures customer satisfaction, cost control, and competitive advantage. Delays can lead to increased expenses and loss of client trust. Project managers can meet deadlines by using Agile practices, breaking down tasks (Work Breakdown Structure), setting clear milestones, using tools like Jira or Trello, performing risk assessments, and holding regular stand-ups or status meetings.
## 2. How does effective cost control contribute to the success of a software project? What strategies can be used to prevent budget overruns?
Cost control keeps the project within budget, improving profitability and resource use. Strategies include defining a clear scope, tracking costs with tools like Microsoft Project or Monday.com, using Earned Value Management (EVM), maintaining a change log, and performing regular financial reviews. Projects typically overrun by 27% on average if not controlled (Standish Group, 2023).
## 3. Compare and contrast Agile and Waterfall methodologies. What are the main advantages and disadvantages of each?
Agile:
- Advantages: Iterative, adaptable, client feedback-friendly, faster releases
- Disadvantages: Harder to estimate cost/time, scope creep risk

Waterfall:
- Advantages: Structured, clear documentation, predictable timelines
- Disadvantages: Inflexible to changes, late testing, less customer involvement
## 4. In what types of projects might Agile be more beneficial than Waterfall, and vice versa? Can you provide examples of each?
- Agile is better for projects with evolving requirements, like mobile apps or e-commerce platforms.
- Waterfall is better for fixed-scope projects like banking systems, aerospace software, or government projects where compliance is critical.
## 5. What are some methods for ensuring quality assurance throughout a software project? Why is it important to maintain high standards?
Methods include Test-Driven Development (TDD), Continuous Integration/Continuous Deployment (CI/CD), peer code reviews, automated testing, and using QA tools like Selenium or Postman. High standards reduce defects, improve user satisfaction, and lower maintenance costs—poor quality can increase project costs by 50% (CIO.com, 2023).
## 6. How does defining the project scope contribute to successful project planning? What is a Work Breakdown Structure (WBS), and why is it useful?
Defining scope avoids scope creep and sets clear goals. A Work Breakdown Structure (WBS) divides the project into smaller, manageable components, enabling better task allocation, resource planning, and tracking. It supports clearer timelines and deliverables.
## 7. What are the benefits of developing a detailed project schedule, and how can Gantt charts assist in this process?
A detailed schedule helps with time management, task prioritization, and resource planning. Gantt charts visually represent task timelines, dependencies, and milestones, improving clarity and accountability.
